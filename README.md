# R7LocalServer

Десктопное приложение — локальный HTTP-сервер для плагинов Р7 Офис.

Плагины Р7 Офис выполняются в браузерном контексте и не имеют прямого доступа к файловой системе и корпоративным сетевым ресурсам. R7LocalServer запускается на рабочем месте пользователя и предоставляет REST API, через который плагины могут читать и записывать файлы, выполнять HTTP-запросы с Windows-аутентификацией и синхронизировать макросы между книгами.

## Запуск

Запустите приложение, выберете подходящий порт, нажмите запустить.

## API

Все запросы — `POST`, тело и ответ — JSON (`Content-Type: application/json`).

---

### Диагностика

#### `GET /ping`

Проверка доступности сервера.

**Ответ:**
```json
{
  "status": "ok"
}
```

### Файлы

Поддерживаемые форматы: `json`, `txt`, `csv`, `xml`, `xls`, `xlsx`.

#### `POST /files/write`

Записывает файл на диск.

**Запрос:**

```json
{
  "directoryPath": "D:\\shared",
  "fileName": "data.json",
  "overwrite": false,
  "content": { }
}
```

| Поле            | Тип     | Обяз. | Описание                                          |
|-----------------|---------|:-----:|---------------------------------------------------|
| `directoryPath` | string  |  ✓   | Абсолютный путь к папке                           |
| `fileName`      | string  |  ✓   | Имя файла с расширением                           |
| `overwrite`     | boolean |       | Перезаписать если существует (по умолчанию false) |
| `content`       | any     |  ✓   | Содержимое файла                                  |

**Ответ:**

```json
{
  "success": true,
  "path": "D:\\shared\\data.json",
  "fileType": "JSON"
}
```

---

#### `POST /files/read`

Читает файл с диска.

**Запрос:**

```json
{
  "directoryPath": "D:\\shared",
  "fileName": "data.xlsx",
  "readMode": "DEFAULT"
}
```

| Поле            | Тип    | Обяз. | Описание                                                 |
|-----------------|--------|:-----:|----------------------------------------------------------|
| `directoryPath` | string |  ✓   | Абсолютный путь к папке                                  |
| `fileName`      | string |  ✓   | Имя файла с расширением                                  |
| `readMode`      | string |       | Режим чтения Excel: `DEFAULT`, `RAW`, `TYPED` (см. ниже) |

**Режимы чтения Excel:**

| Режим     | Описание                                                                         |
|-----------|----------------------------------------------------------------------------------|
| `DEFAULT` | Автоматический выбор: значения типизированы по содержимому                       |
| `RAW`     | Возвращает сырые строки без конвертации типов                                    |
| `TYPED`   | Возвращает ячейки с явно указанным типом (`string`, `number`, `boolean`, `date`) |

**Ответ:**

```json
{
  "directoryPath": "D:\\shared",
  "fileName": "data.xlsx",
  "fileType": "XLSX",
  "lastModifiedAt": "2024-01-15T10:30:00",
  "content": { }
}
```

### Excel

#### `POST /export/excel-sheets`

Записывает листы с данными в Excel-файл. Читает шаблон из источника, заполняет указанные листы и сохраняет результат.

**Запрос:**

```json
{
  "sourceDirectoryPath": "D:\\templates",
  "sourceFileName": "template.xlsx",
  "targetDirectoryPath": "D:\\output",
  "targetFileName": "report.xlsx",
  "overwrite": false,
  "sheets": [
    {
      "name": "Лист1",
      "cells": [
        { "row": 0, "col": 0, "value": "Текст" }
      ]
    }
  ]
}
```

| Поле                  | Тип     | Обяз. | Описание                        |
|-----------------------|---------|:-----:|---------------------------------|
| `sourceDirectoryPath` | string  |  ✓   | Папка с шаблоном                |
| `sourceFileName`      | string  |  ✓   | Имя файла шаблона               |
| `targetDirectoryPath` | string  |  ✓   | Папка для сохранения результата |
| `targetFileName`      | string  |  ✓   | Имя результирующего файла       |
| `overwrite`           | boolean |       | Перезаписать если существует    |
| `sheets`              | array   |  ✓   | Листы с ячейками для записи     |

**Ответ:**

```json
{
  "targetDirectoryPath": "D:\\output",
  "targetFileName": "report.xlsx",
  "exportedSheetNames": ["Лист1"]
}
```

### Прокси

#### `POST /proxy/request`

Выполняет HTTP-запрос от имени сервера. Поддерживает аутентификацию, включая NTLM и Negotiate (Kerberos) для корпоративных ресурсов.

**Запрос:**

```json
{
  "url": "https://intranet.company.local/api/data",
  "method": "GET",
  "headers": { "Accept": "application/json" },
  "body": null,
  "authType": "NTLM",
  "username": "user",
  "password": "pass",
  "domain": "CORP",
  "ignoreSslErrors": false
}
```

| Поле              | Тип     | Обяз. | Описание                                                 |
|-------------------|---------|:-----:|----------------------------------------------------------|
| `url`             | string  |  ✓   | URL запроса                                              |
| `method`          | string  |       | HTTP-метод (по умолчанию `GET`)                          |
| `headers`         | object  |       | Заголовки запроса                                        |
| `body`            | string  |       | Тело запроса                                             |
| `authType`        | string  |       | Тип аутентификации: `NONE`, `BASIC`, `NTLM`, `NEGOTIATE` |
| `username`        | string  |       | Логин                                                    |
| `password`        | string  |       | Пароль                                                   |
| `domain`          | string  |       | Домен Windows (для `NTLM`)                               |
| `ignoreSslErrors` | boolean |       | Игнорировать ошибки SSL-сертификата                      |

**Ответ:**

```json
{
  "status": 200,
  "headers": { "Content-Type": "application/json" },
  "body": "..."
}
```

### Синхронизация макросов

#### `POST /macros/sync`

Управляет библиотекой универсальных макросов, общей для всех книг. Библиотека хранится как JSON-файл на диске.

**Запрос:**

```json
{
  "mode": "push",
  "directoryPath": "D:\\shared",
  "fileName": "universal_macros.json",
  "macrosArray": [],
  "selectedGuids": []
}
```

| Поле            | Тип      | Обяз. | Описание                                                  |
|-----------------|----------|:-----:|-----------------------------------------------------------|
| `mode`          | string   |  ✓   | Режим: `push`, `merge`, `load`, `refresh`, `delete`       |
| `directoryPath` | string   |  ✓   | Путь к папке с файлом библиотеки                          |
| `fileName`      | string   |  ✓   | Имя файла библиотеки                                      |
| `macrosArray`   | array    |       | Макросы текущей книги (не требуется для `delete`)         |
| `selectedGuids` | string[] |       | GUID-ы макросов для `load` и `delete`                     |

**Режимы:**

| Режим     | Описание                                                                                                                                                                                |
|-----------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `push`    | Публикует макросы с `isUniversal: true` из книги в общий файл. Обновляет существующие по GUID, добавляет новые. Создаёт файл если не существует.                                        |
| `merge`   | Объединяет общий файл с книгой: универсальные макросы идут первыми, затем разделитель, затем специфичные для книги. Возвращает готовый `macrosArray`.                                   |
| `load`    | Загружает выбранные (`selectedGuids`) макросы из общего файла в книгу. Заменяет существующие по GUID, новые добавляет в начало. Возвращает список конфликтов для диалога подтверждения. |
| `refresh` | Обновляет только те макросы книги, чьи GUID есть в общем файле, не добавляя новых. Используется для авто-синхронизации при открытии документа.                                          |
| `delete`  | Удаляет макросы с указанными `selectedGuids` из общего файла. Книгу не изменяет.                                                                                                        |

**Ответ:**

```json
{
  "status": "ok",
  "macrosArray": [],
  "conflicts": ["Macro A"],
  "updated": ["Macro B"],
  "added": ["Macro C"],
  "deleted": ["Macro D"],
  "totalUniversal": 5
}
```

| Поле             | Режимы                     | Описание                                      |
|------------------|----------------------------|-----------------------------------------------|
| `macrosArray`    | `merge`, `load`, `refresh` | Готовый массив макросов для записи в документ |
| `conflicts`      | `load`                     | Макросы, которые будут перезаписаны           |
| `updated`        | все кроме `delete`         | Имена обновлённых макросов                    |
| `added`          | `push`, `load`             | Имена добавленных макросов                    |
| `deleted`        | `delete`                   | Имена удалённых из файла макросов             |
| `totalUniversal` | `push`, `delete`           | Итоговое количество макросов в общем файле    |

## Плагины Р7 Офис

В репозитории находятся готовые плагины для Р7 Офис (папка `plugins/`), использующие API R7LocalServer.

### Установка

1. Скачайте плагины из релиза
2. Откройте Р7 Офис → **Плагины** → **Настройки** → **Добавить плагин**.
3. Укажите файл плагина (`Macros Sync.plugin` или `Macros Sync Companion.plugin`).
4. Повторите для каждого плагина.

Для автосинхронизации при открытии книги нужны **оба** плагина.

---

### Macros Sync

Визуальный плагин для управления библиотекой универсальных макросов. Открывается через вкладку **Плагины**.

**Настройки** (раскрывающаяся панель):

| Поле                              | Описание                                                                                       |
|-----------------------------------|------------------------------------------------------------------------------------------------|
| Адрес сервера                     | URL R7LocalServer (по умолчанию `http://127.0.0.1:8124`)                                       |
| Папка с общим файлом              | Путь к папке, где хранится файл библиотеки макросов                                            |
| Имя файла                         | Имя JSON-файла библиотеки (по умолчанию `universal_macros.json`)                               |
| Папка резервных копий             | Путь для сохранения резервной копии при авто-синхронизации (необязательно)                     |
| Авто-синхронизация при открытии   | Включает/отключает авто-синхронизацию через плагин Macros Sync Companion                       |

**Левый столбец — «Текущая книга»:**

Показывает все макросы открытой книги. Для каждого макроса доступны два чекбокса:

- **универсальный** — пометить макрос как универсальный. Универсальные макросы отображаются вверху списка, над разделителем. При публикации попадают в общий файл.
- **не синхр.** _(только для универсальных)_ — исключить из автоматической синхронизации. Companion не будет перезаписывать этот макрос при открытии книги.

Кнопка **«Опубликовать универсальные»** записывает все макросы с пометкой «универсальный» в общий файл (`mode=push`).

**Правый столбец — «Сохранённые»:**

Показывает макросы из общего файла библиотеки. Значок **«есть»** означает, что макрос уже присутствует в текущей книге.

- Чекбоксы у каждого макроса и кнопка **«Все»** — выбор макросов.
- **«Загрузить выбранные»** — добавляет выбранные макросы в книгу (`mode=load`). Перед перезаписью показывает список конфликтов.
- **«Удалить из файла»** — удаляет выбранные макросы из общего файла (`mode=delete`). Книгу не изменяет.

### Macros Sync Companion

Фоновый системный плагин. Не имеет интерфейса, запускается автоматически при каждом открытии книги.

**Что делает:** если в настройках включена «Авто-синхронизация при открытии», при открытии книги сравнивает GUID-ы её макросов с общим файлом и обновляет код тех, что там есть (`mode=refresh`). Макросы с флагом **«не синхр.»** не перезаписываются.

**Резервное копирование:** если указана папка резервных копий, перед синхронизацией сохраняет текущие макросы книги в файл `universal_macros_backup.json`.

Плагин использует те же настройки из `localStorage`, что и Macros Sync (ключи `macrosSync_*`), поэтому настраивать нужно только один раз через основной плагин.

---

При ошибке сервер возвращает соответствующий HTTP-статус и JSON:

```json
{
  "status": 400,
  "message": "File already exists"
}
```

| Статус | Причина                                                                                    |
|--------|--------------------------------------------------------------------------------------------|
| `400`  | Некорректный запрос: пустые поля, неверный режим, файл уже существует и `overwrite: false` |
| `404`  | Файл не найден                                                                             |
| `500`  | Внутренняя ошибка сервера                                                                  |
